## Привязка брейкпоинтов CSS ↔ ширины макета

В `production.css` по сути используются:

| Условие | Ближайший фрейм Figma |
|---------|------------------------|
| **`≥1441px`** | ближе всего к **1920** |
| **`1280–1440px`** | **1440** |
| **`992–1279px`** | ближе к **1024** (отдельного порога «ровно 1024» нет) |
| **`768–991px`** | **768** (планшет) |
| **`≤767px`** и **`≤479px`** | мобила / узкая мобила (**390** логично тестировать при `≤479` и `≤767`) |

Отдельного правила «ровно под 1024» нет — этот кадр попадает в **992–1279** или в пересечения с другими блоками.

## Построчное сопоставление контента и структуры

### Hero (`#Hero-Section`)

**Баннеры:** в HTML **4** пункта, как на **1920 / 1440 / 768**:

1. `#` + «8 лет» / «в рекламе»  
2. `#` + «17k+» / «креативов»  
3. `#` + «100+» / «музыкальных видео»  
4. «Full-cycle» / «ai продакшн»  

- **1024 в Figma:** четвёртая пара `#Full-cycle` / «Видеоклипы» — **с сайтом не совпадает** (нужна замена через responsive-копи или правка макета).  
- **390 в Figma:** 3 колонки, другие подписи («Концепт-Арты», нет 100+) — в HTML **один вариант для всех ширин**.

**Подзаголовок:** в HTML есть «Современный» в `.hero-modern-prefix`; на эталонном **1920** в тексте этого не было — обычно добивается **CSS** (`display: none` у префикса), не вычищая разметку.

### Клиенты (`section.client-section`)

Структура совпадает с макетами (полоса + маркиз + градиенты). Детали набора логотипов по ширинам — только визуально в Figma.

### Проекты (`#Project-Section`)

- Заголовок и абзац под ним совпадают с **1440 / 1024 / 768** (один блок описания).
- **Карточка «Для артистов»:** в HTML «AI-клипы и сниппеты…»; в **1440 / 1024 / 768** в Figma было «Полноценные ai клипы…» — **расхождение копирайта**.
- **Карточка «Для брендов»:** HTML с точкой в конце; в **1024** в метаданных точки не было — мелочь.
- Третий блок — видео `lisacreo-eyes.mp4` внутри `.project-block`, как на макетах с полным рядом.

### Видеопродакшн (галерея)

> **Примечание:** ниже описано состояние, которое фиксировалось при сравнении с макетом. В текущей рабочей копии секция могла быть уже доработана (заголовок `# Видеопродакшн`, `id`, стили в `production.css`) — сверяйте с актуальным `index.html`.

- В одном из состояний: `.gallery-section` **без** заголовка «# Видеопродакшн» и без подзаголовка — только сетка из трёх `.gallery-block`. Во фреймах Figma эта секция **с заголовком**.
- По количеству карточек: совпадает с **1920 / 1440 / 1024** (три). На **768** в Figma две колонки — задача **CSS** (`display: none` третьей или перестройка сетки). На **390** — одна плитка; три узла в DOM могут оставаться — снова **CSS/адаптив**.

### Услуги (`#Service-Section`)

- В HTML три `.service-block` (AI-фото, AI-видео, Креативы) — как **1920 / 1440 / 1024**.
- В **768** в Figma только **две** карточки — для pixel-perfect нужно скрывать третью или менять сетку на планшете.
- Дополнительно на сайте есть блок `.sectitle-right-box`: подзаголовок «От стратегии до визуала…» и кнопка «Обсудить проект» — на **1920** заголовок секции был только «# Чем я могу помочь» без этой правой колонки в том виде. Это **расширение относительно макета**.

### CTA (`section.call-to-action`)

- Текст совпадает с **1440**: «Готовы создать что-то крутое?» (**без запятой** после «создать»), как в HTML.
- На **1920** в другом фрейме могла быть версия **с запятой** — расхождение **между двумя фреймами Figma**, не обязательно между сайтом и 1440.
- Подпись под заголовком: в HTML **без точки** в конце; на **1024** в Figma **с точкой** — при желании подогнать одну строку через `@media`.

### Визуалы (`section.marquee-section`)

- В HTML подпись `p.section-subtitle` «Визуалы» и `h2` — одна длинная фраза (без отдельного `#` в разметке); в Figma обычно **# Визуалы** отдельным заголовком и подпись второй строкой. Семантика близка, **структура заголовков отличается**.
- Набор колонок маркиза богаче, чем в отдельных фреймах — ок, если так задумано.

### Контакты (`#Contact-Section`)

В актуальном `index.html` якорь **`#Contact-Section`** стоит на **`<footer class="main-footer">`** (колонки меню/услуги/соцсети и т.д.), отдельной секции **`section.contact-section`** с формой на главной может не быть — сверять с файлом. В макете **768** контакты могут быть отдельным крупным блоком с формой.

### Футер

В HTML **© 2025**; в **1920** в макете могло быть **2026**, в **768 / 390** снова **2025** — сверить с брендом, какой год финальный.

## Сводная таблица: HTML vs Figma

| Секция | Реальность в HTML/CSS | Дельта с Figma | Подсказка |
|--------|------------------------|----------------|-----------|
| Hero, 4-й баннер | «Full-cycle» / «ai продакшн» | 1024: в макете «Видеоклипы» | Синхронизировать копирайт или Figma |
| Hero, узкая ширина | В DOM всегда 4 блока | `≤767px`: CSS скрывает 3-й баннер → на экране 3 тега | 390 в макете «3 баннера» согласуется с CSS, не с «урезанным» HTML |
| Hero «Концепт-Арты» | Отдельного баннера нет | В 390 макет мог показывать отдельный баннер | Либо макет, либо прод |
| Галерея | 3 блока / 3 картинки | `main.css`: при `≤991` часто 2 колонки, при `≤479` — 1; в `production.css` могли быть оверрайды (#Gallery-Section) | Сверять актуальные правила в обоих файлах |
| Услуги | 3 карточки | `main.css`: `≤991` — 2 колонки, `≤767` — 1 | 768 в Figma «2 карточки» — решить судьбу третьей |
| CTA заголовок | Без запятой | Разные фреймы Figma могут отличаться | Выбрать канон |
| Marquee | «Визуалы» + длинный `h2` | Структура `#` может отличаться | Сверить с конкретным узлом Figma |
| Галерея vs название | Проверить наличие заголовка/id | В макете всегда «Видеопродакшн» | a11y / якоря |

## `production.css`: `@media` и соответствие фреймам

Файл: `css/production.css`

| Условие | Смысл / комментарии в файле |
|---------|------------------------------|
| `(min-width: 1441px)` | Широкий десктоп; отступы «большого» макета |
| `(min-width: 1280px)` | Hero без префикса «Современный», типографика под крупный экран |
| `(min-width: 1280px) and (max-width: 1440px)` | Явно: Figma **1440** |
| `(min-width: 992px) and (max-width: 1279px)` | Явно: Figma **992** |
| `(min-width: 768px) and (max-width: 991px)` | Планшет **768** |
| `(max-width: 1279px)` | Общие правки hero и хедера |
| `(max-width: 991px)` | Поведение планшета и ниже (часть правил) |
| `(max-width: 767px)` | Мобила; скрытие 3-го баннера и др. |
| `(max-width: 479px)` | Узкая мобила (**390**) |
| `(hover: hover)`, `(prefers-reduced-motion: reduce)` | Не привязаны к ширине фрейма |

**Дополнительно:** `css/main.css` — например `max-width: 991px`: галерея **2** колонки, услуги **2** колонки; `max-width: 767px`: галерея **2** колонки, услуги **1**; `max-width: 479px`: галерея **1** колонка. Оверрайды в `production.css` для `#Gallery-Section` могут менять это поведение — проверять оба файла.

**Итог по брейкпоинтам:** **1024** как отдельный порог в `production.css` не выделен (попадает в **992–1279** или общие `≤1279`). **768** — связка **768–991** и **≤767**. **390** — логично сверять с **`≤479`** + типографикой из **`≤767`**.
